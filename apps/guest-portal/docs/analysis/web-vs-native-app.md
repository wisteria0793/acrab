# Webアプリ vs ネイティブアプリ 比較分析 (Web Service vs Native App)

ゲストポータルを「ブラウザで開くWebサービス」にするか、「アプリストアからダウンロードするネイティブアプリ」にするかの比較・分析です。

## 結論 (Conclusion)
**Webサービス (PWA) が圧倒的に推奨されます。**
短期滞在（数日）のために数十MBのアプリをダウンロードさせることは、ゲストにとって大きなストレス（摩擦）となり、利用率を極端に下げます。

---

## 比較表 (Comparison)

| 特徴 | Webサービス (Web App / PWA) | ネイティブアプリ (iOS / Android) |
| :--- | :--- | :--- |
| **アクセス (Friction)** | **低**: QRスキャン → 即開く (1秒) | **高**: ストア移動 → DL → インストール (数分) |
| **開発コスト** | **低**: 単一コードベース (Next.js) | **高**: iOS/Android両対応が必要 |
| **更新の容易さ** | **即時反映**: サーバー更新のみ | **審査待ち**: Apple/Googleの審査が必要 (数日) |
| **通知機能** | △: ブラウザ通知 (iOSは制限あり) | **◎**: プッシュ通知が確実に届く |
| **オフライン** | ○: Service Workerで一部対応可 | **◎**: フル機能で対応しやすい |
| **ゲスト心理** | 「チェックインのためなら開く」 | 「たった2日のためにアプリ入れたくない」 |

## 詳細分析

### 1. ユーザーの摩擦 (User Friction)
*   **Web**: カメラでQRをかざすだけで起動します。この「手軽さ」が、チェックイン完了率を最大化します。
*   **Native**: 「ストアのパスワード忘れた」「容量がない」「ギガが減る」などの理由で、ダウンロードを拒否されるケースが多発します。特に海外ゲストはローミング中の場合があり、大容量通信を嫌がります。

### 2. PWA (Progressive Web App) という選択肢
「アプリのような使い勝手」をWebで実現する技術です。
*   **ホーム画面に追加**: アイコンを置くことができます。
*   **オフライン動作**: キャッシュにより、電波がなくても画面を表示できます。
*   **開発**: 現在のNext.jsでそのまま対応可能です。

## 推奨方針 (Recommendation)

**「Webファースト」で行くべきです。**

1.  **基本**: QRコードからブラウザでアクセスさせる。
2.  **アプリ化**: もしリピーターや長期滞在者にアプリを使わせたい場合でも、まずはPWAとして「ホーム画面への追加」を促すだけで十分機能します。

ネイティブアプリ開発は、数十店舗を展開するチェーンホテル等の規模になってから検討するフェーズの話です。個人〜小規模の民泊ではROI（費用対効果）が悪すぎます。
